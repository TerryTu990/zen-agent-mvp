# xianyu-orders 功能事实

> 2026-07-21 登录态真机只读验证；不含账号、买家、商品或订单真值。

- 订单管理 hash 路由为 `#/seller-trade/order-manage`。
- 订单详情从“查看详情”打开独立标签页，hash 路由为 `#/seller-trade/order-manage/order-detail?orderId=<订单号>`。
- 状态筛选包括：全部、待付款、待发货、发货即将超时、发货已超时、已发货、售后中、交易成功、交易关闭。
- 列表表头包括商品信息、发货/退款状态、单价/数量、成交价、买家/收货信息、操作。
- 每个订单由摘要行与商品明细行组成：摘要行展示平台状态、订单编号和下单时间；明细行展示商品、状态、买家联系入口和查看详情。
- “联系ta”入口的 hash 路由携带 `itemId`、`orderId`、`peerUserId`；进入后仍需重新快照核对目标。
- 待发货计数为零时，选择该状态后表格显示“暂无数据”。
- 当前侦察未发现 iframe、Shadow DOM、虚拟列表或验证码；该结论只覆盖列表第一页与一条已发货订单详情。
- 已发货订单详情可见订单编号、商品链接和步骤状态；商品链接形如 `https://www.goofish.com/item?id=<商品号>`。
- 2026-07-22 侦察时待发货计数仍为零，因此尚未在生产页面观察“发货”按钮点击后的确认弹窗；自动路径只允许受控单击并以新快照确认“已发货”，出现额外弹窗即转人工。
