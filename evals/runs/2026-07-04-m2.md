## 三、M2 资产改动走查结论（ZA-C-EVAL-01 / -02 / -03）

本期涉 ZA-EVAL 走查的资产改动（`git diff` vs HEAD=b22ee28，工作树未提交）：

1. `examples/host-demo/config/features/order-list/facts.md`：+1 行登记「导出按钮：`#btn-export`，导出当前订单列表为文件」——新增引导锚点。
2. `examples/host-demo/order-list.html`：+1 行 `<button id="btn-export">导出</button>`——facts 锚点在宿主页有对应真实 DOM 元素（命中路 DOM 前提）。
3. `evals/scenarios.json`：+2 场景 m2-guide-01（命中）/ m2-guide-02（失配）——引导命中维度落地。
4. `examples/host-demo/config/features/order-list/feature.md`：**本期未改**（`git diff` 无该文件）；引导规则 ZA-FEAT-02「引导只用登记锚点」M1 即存在，M2 复用，无新增/删除规则条目。

触发分层（ZA-C-EVAL-02）：改动落在**单 feature（order-list）配置 + 其宿主页 + 评测集**，非基座 system-prompt，
按"改单 feature 跑该 featureId 子集"本可只跑 order-list 引导子集；但本期同时新增了装配/网关引导消费实现（gateway 工具注入门、page-action），
故按更严口径已跑**全量单测 + M1 回归 E2E + M2 引导 E2E**（见第一节），覆盖充分。

走查判据与结论：
1. **引导命中链未臆造 selector**：命中路 `selector` 恒取自 facts 登记锚点 `#btn-export`；服务端不校验 selector 是否真在 facts 内，
   失配由客户端 DOM 命中与否兜底（`gateway.ts` 注释所述），DOM 命中是事实、非治理判定（符合边界铁律：引导不引入客户端治理判定）。→ ✅
2. **失配路如实降级、不假装高亮**：guide-02 三层证据一致——不下发 guide-action 帧、无元素高亮、无成功文案、出现如实降级文本。→ ✅ 无"假装定位"污染。
3. **引导为终结动作、不经 toolgate**：`gateway.ts` 命中 `guide_highlight` 直接 `broadcast(guideFrame(...))` 后 `break`，本回合结束、
   **不回喂 observation**（server 集成断言 `textOf(sse.frames)===''`）；guide 工具不入 tools.json、不经 toolgate（那是 M3 宿主 API 工具路径）。→ ✅ 符合 D9 与边界铁律。
4. **page-action 闭集（D9）**：`apps/extension/src/page-action.ts` 触达面仅 `highlight`/`unhighlight`/`scrollIntoView`，
   action 闭集 `highlight|scroll-to`（contracts schema 越集校验 `schemas.test.ts:119`），只做视觉标记与滚动，无填表/点按等 DOM 替点。→ ✅
5. **示例互斥（ZA-C-EVAL-03）**：走查 `order-list/feature.md`、`facts.md` 未内嵌与 m2-guide-01/02 重叠的问答示例
   （facts 为元素锚点条目、feature 为规则条目，非示范问答），无评测污染。→ ✅
6. **≥3 跑回归（ZA-C-EVAL-02）**：mock 为确定性实现（`#btn-export`+"导出"双条件触发），通过率方差为 0，单跑即确定，
   引导装配/失配降级存在性判定不需重复采样；`≥3 跑通过率比较`针对**真实 LLM 输出方差**，锚点 = M4 评测 runner。
   M2 引导 E2E 已额外独立复跑一次（两次 exit 0）佐证确定性。

结论：本期 M2 引导为**新增能力**（未改动 M1 已验证讲解/拒答/换出语义），全量门（build / 114 单测 / lint:deps / M1 回归 E2E / M2 引导 E2E）全绿，
guide 维度命中/失配两场景走查证据齐全且无退化；引导装配链路、终结动作不回喂、page-action 闭集（D9）、失配如实降级客观成立；
真实 LLM 引导选锚点质量与降级措辞的量化评测（含 ≥3 跑通过率）锚点 = **M4 评测 runner 接真实 provider**。
