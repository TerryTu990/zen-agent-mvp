## 三、三反例分层（U7 安全模型真正被测处）

建设提示词 Goal-d "另验：拒绝分支、nonce 重放被拒、forbidden fail-closed"——按可观察性分层，**均有测试兜底**：

| 反例 | 覆盖层 | 证据 |
|---|---|---|
| 拒绝分支 | 浏览器 E2E d2 | HITL 卡片点拒绝 → user-rejected 回喂、无代执行 |
| forbidden fail-closed | 浏览器 E2E d3 + toolgate 单测 + server 集成 | E2E DELETE 计数 0；toolgate `riskTier=forbidden → deny forbidden`；server `forbidden 工具 → 服务端 deny、无 HITL/无 exec-instruction` |
| nonce 重放被拒 | toolgate 单测 + server 集成 | toolgate `同 nonce 二次回收 → replayed（一次性防重放）`；server `exec-result 重放（同 nonce 二次 POST）→ 409、不再入 toolgate、不重复执行` |
| ttl 超时 | toolgate 单测 | `超过 ttl → timeout 并核销` |
| invalid-result（不采信上报） | toolgate 单测 | `ok:true 但 body 不过 resultSchema → invalid-result` |
| 伪造 nonce | server 集成 | `伪造 nonce 的 exec-result → 409（无对应挂起回合）` |

重放/ttl/invalid-result 归 toolgate 单测 + server 集成而非浏览器 E2E：这些是服务端一次性签名核销的确定性判定，
浏览器层无法自然构造（客户端零治理、不持签名密钥），在服务端层测试更确定、更贴近信任边界。

## 五、U7 安全模型自审（交叉校验要点）

- **决策服务端 fail-closed**：分级/HITL 判定只在 `packages/toolgate`；unknown-tool/riskTier未知/channel非client/invalid-params/identity/forbidden 六路 deny，均有单测。gateway.decide 带 C2 claims。
- **一次性签名**：issueExecInstruction 生成 nonce+ttl+HMAC 签名并注册；acceptExecResult 核销 nonce（二次=replayed）、验 ttl（过期=timeout）。gateway `waitForExec`/`waitForHitl` 在广播帧前注册等待器（无竞态丢帧）。
- **结果必过校验才回喂**：acceptExecResult 按工具 resultSchema 校验 body，不过=invalid-result、content=null，不采信客户端上报原文。
- **客户端零治理**：extension delegated-executor 只按已定值请求 fetch、不验 riskTier/不解析工具语义、signature 不作客户端准入；下行 exec-instruction 含 signature/nonce 属正常下发。
- **SEC**：signingSecret/jwtSecret 只经 env 注入（main.ts requireEnv，缺失拒启）；tool-card summary 仅 toolId 不含实参；错误文案不含密钥/token 原文。
- **端口冻结**：observation 经 role:tool 回喂，未改 LlmMessage/ports.ts/schemas；真实 provider 的 assistant.tool_calls 回声限制已挂锚点（接真实 provider 时评估）。
