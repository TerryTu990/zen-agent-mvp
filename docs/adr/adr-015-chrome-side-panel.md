# ADR-015：Chrome Side Panel 承载持久对话，页面仅承担观察与执行

## 状态

已接受（2026-07-21）。ADR-014 已由 ADR-013 预留给用户记忆，因此本决策使用 015。

## 背景

现有 Chrome 扩展把完整对话抽屉注入每个已激活页面。跨标签页和跨站导航时，UI 生命周期与页面生命周期绑定，容易出现重复 UI、对话镜像和页面刷新后重建。Zen Agent 后续要在一个任务中持续操作多个非自研站点，需要把“用户持续面对的对话”与“当前被授权执行的页面”分开。

## 决策

- 使用 Chrome 原生 Side Panel 承载对话、工具进度、HITL、停止入口和执行偏好。
- Side Panel 是 Chrome 客户端内部实现，不为服务端 C3 五能力契约增加形态字段。
- background 是会话、SSE 和路由权威；Side Panel 不直接连接服务端。
- content script 不再渲染对话，只负责上下文、快照、引导、页面动作和代执行。
- Side Panel 的全局持续不授予任意标签页执行资格。执行只路由到任务组内的权威活动页；任务组外页面仅显示为不可执行。
- 最终产物不保留网页抽屉或第二套会话 UI。

## 理由

Chrome Side Panel 的生命周期天然独立于网页导航，能让用户在站点切换时保持同一操作上下文；把执行页资格继续交给既有任务组，可避免“看见页面”等同于“获准执行”的权限扩大。服务端仍只处理现有会话帧，符合接入形态差异不外泄的约束。

## 后果

- 扩展内部增加 Side Panel 与 background 的私有消息路由。
- E2E 需要同时驱动 Side Panel 页面和宿主页面。
- Chrome 之外的浏览器不在产品支持范围。
