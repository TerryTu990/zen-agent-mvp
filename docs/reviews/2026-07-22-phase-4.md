## 发布中发现并修复的问题

1. legacy 基线登记与激活脚本在单引号 Go template 中错误转义双引号，Docker 拒绝解析挂载模板；`2e70ddb` 修复两处模板并增加静态契约，Linux 回滚行为测试通过。
2. 首次旧镜像早于外置 `system-prompt.md` 和 `lark-cli` 契约，健康旧版被新验证器误拒；`04d3c02` 为首次登记 descriptor 增加明确 `ZA_RELEASE_LEGACY=1`，只对该 descriptor 放宽后来新增的两项，继续强制健康、单副本、镜像、快照与数据卷。Linux 行为测试新增直接 legacy 目标场景并通过。
3. 第一次真实旧版演练因 CLI 新契约失败，链接未切换但容器已短暂运行旧版；主流程立即恢复 `2e70ddb` 并验证公网健康。兼容修复发布后，第二次旧版演练完整通过并恢复 `04d3c02`。

## 生产启用边界

当前状态为 `deployed-disabled`，不是 `production-accepted`：

- 生产飞书 `general` profile：未授权。
- `ZA_FEISHU_CARD_BASE_TOKEN` / `ZA_FEISHU_CARD_TABLE_ID` / `ZA_FULFILLMENT_GUIDE_URL`：未同时启用。
- `ZA_FULFILLMENT_PRODUCT_KEYS_JSON`：未配置。
- `ZA_FULFILLMENT_POLICIES_JSON`：disabled；shipment 与 delivery 均无生产预批准策略。
- Chrome 自动扫描默认关闭；没有触发任何真实闲鱼订单副作用。

启用前必须由 Terry 在服务器受控卷完成飞书用户授权，并在服务器 `.env` 填入真实业务坐标、商品映射及两条独立有界策略；值不得经开发机或仓库中转。重建容器后需重新跑 whoami、Base schema/只读查询、单笔低价值订单验收，才可改为 `production-accepted`。
