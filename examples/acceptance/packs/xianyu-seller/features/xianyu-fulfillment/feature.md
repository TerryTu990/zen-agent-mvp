# xianyu-fulfillment 功能规则（编号 `ZA-FEAT-NN`）

你当前位于闲鱼买家联系页。消息编辑器与发送控件已完成只读侦察，但真实发送回显和验证码形态尚未验证。

## ZA-FEAT-01 只允许非秘密测试占位内容

`xianyu-fulfillment.compose-test-message` 仅用于用户本次明确提供的非秘密测试占位内容。真实卡密、token、密码或其他 secret MUST NOT 进入工具参数、模型上下文或快照；不得用此工具发送真实卡密。

## ZA-FEAT-02 订单身份必须重新绑定

准备测试消息前，必须从当前 hash 路由与页面证据重新核对 `orderId`，且与订单页确认的目标一致；不能仅凭买家昵称或商品名称绑定订单。

## ZA-FEAT-03 填写与发送分离

填写非秘密测试内容走 `compose-test-message`；点击发送必须走 `send-test-message`。发送工具为 `every-call`，每次都由平台独立确认，不得把点击发送夹在准备批次里。

## ZA-FEAT-04 不确定结果禁止重发

发送后 MUST 再次 `page_snapshot`，将“页面明确成功”“明确失败”“结果不明确”分开处理。输入框清空或执行结果 `ok` 单独都不是成功证据；结果不明确时停止并转人工，禁止自动重复发送。

## ZA-FEAT-05 当前不更新发货状态

当前只验证消息发送闭环，不点击发货、提醒收货、延长收货或其他订单状态动作。真实发送成功回显未完成测试前，不得声称订单已履约。
