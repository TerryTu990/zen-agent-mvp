# resume 功能规则（编号 `ZA-FEAT-NN`）

你当前辅助的功能是 BOSS直聘 求职者端的"我的简历"页：读取用户简历以了解其求职画像（工作经历、教育、期望岗位/城市/薪资、技能），供本次求职任务的职位筛选与沟通起草使用。

## ZA-FEAT-01 讲解与操作以功能事实为准

讲解本页结构或在页面上代操作时，只依据本功能事实（facts）；facts 未覆盖的隐私设置、简历模板、附件等内容如实说明"当前功能配置未覆盖"，不臆造元素或入口。

## ZA-FEAT-02 只读采集画像，不修改简历、不臆造经历

用 `resume.page-operate` 的 `read` 步骤采集简历字段形成会话画像；MUST NOT 编辑、删除或新增任何简历内容（本功能仅读取）。采集到的画像 MUST 忠实于页面所载，MUST NOT 虚构或夸大用户的经历、技能与期望。

## ZA-FEAT-03 画像是会话数据，不落任何持久制品

采集到的求职画像仅用于当前会话任务；MUST NOT 写入配置、日志或事实文件；跨会话不留存——后续任务如需画像，重新读取本页。

## ZA-FEAT-04 先快照后动作，以页面证据为准

使用前必须先调用 `page_snapshot` 获取可交互元素清单，`read` 步骤的 ref 都必须取自最近一次快照；元素定位靠可见文本 / aria-label / 角色，勿依赖动态生成的 id。分区信息若需展开"查看更多"才完整，先展开再采集。
