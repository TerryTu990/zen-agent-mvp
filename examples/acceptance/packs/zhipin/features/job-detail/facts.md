# job-detail 功能事实

> 部分锚点已于真实登录态核实；仍标 `⚠待核` 者未经真实交互验证，不得当作已知事实向用户陈述。

## 页面构成

- **主路径：列表页 `/web/geek/jobs` 右侧内嵌详情面板**——点某职位卡片（`.job-card-wrap`）即在右侧渲染该职位完整详情
  （职位名 / 薪资 / 城市·经验·学历 / 「职位描述」含岗位职责 / 公司信息 / 「立即沟通」）。**匹配评估与打招呼都可在此完成，无需进独立详情页。**
- 独立职位详情页 `⚠待核`（雏形 `/job_detail/...`；点卡片默认走内嵌面板、未跳独立页，独立页入口待核）。

## 可读取用于匹配评估的字段

- 职位职责、任职要求（硬性条件如经验年限、学历、英语、技能栈、年龄）、工作地点、薪资区间、公司规模/性质。
- 这些是 `skills/jd-match` 硬淘汰（H1-H6）与软评分（S1-S4）、回复概率（R1-R3）判定的完整输入。

## 交互锚点

- 内嵌详情面板顶部动作条含：收藏 / **「立即沟通」/ 举报**；「立即沟通」按钮为 `<a class="op-btn op-btn-chat">`（文案「立即沟通」）。
  打招呼（`job-detail.greet` / `job-search.greet`）click 步骤 ref 一律取自最近一次 `page_snapshot`，不写死选择器（此 class 仅供识别）。
- 「职位描述」区块（`[class*="job-detail"]` / `[class*="job-sec"]`）含岗位职责与要求正文；任职要求常并入「职位描述」，不一定有独立「任职要求」标题。
- ⚠ 待核：是否存在独立「投递简历/正式申请」入口——正式投递走 `job-detail.formal-apply`（every-call），与「立即沟通」打招呼区分；当前内嵌面板只见「立即沟通」。
- ⚠ 待核：点「立即沟通」后是原地保留还是跳转聊天页（对外不可撤回，留到打招呼授权阶段核实）。
