# job-detail 功能事实

> ⚠ 待核：以下页面路径与元素锚点为雏形占位，须在真实 zhipin 页面核对后填实（触发锚点：接入真实 zhipin 页面做首轮评测时）。

## 页面构成

- 职位详情页（路径约 `/job_detail/...`）：职位名 / 薪资区间 / 职位职责 / 任职要求 / 工作地点 / 公司信息 / "立即沟通"按钮。

## 可读取用于匹配评估的字段

- 职位职责、任职要求（硬性条件如经验年限、学历、技能栈）、工作地点、薪资区间。

## 交互

- 用 page-operate 的 `read` 步骤（带 `name`）采集职责与任职要求文本，供对照画像评估；用 `highlight` 指向关键要求。
- ⚠ 待核：任职要求是否折叠、公司信息是否异步加载。
- 点"立即沟通"打招呼走 `job-detail.greet`（判定达标后，一次点击，系统自动发默认招呼语）；`page-operate` 只读不点。
- ⚠ 待核：点"立即沟通"后是原地保留详情页、还是跳转聊天页——影响连续对多个职位打招呼的走法。
