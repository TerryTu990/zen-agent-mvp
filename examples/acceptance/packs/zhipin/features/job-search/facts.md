# job-search 功能事实

> ⚠ 待核：以下页面路径与元素锚点为雏形占位，须在真实 zhipin 页面核对后填实（触发锚点：接入真实 zhipin 页面做首轮评测时）。

## 页面构成

- geek（求职者）端职位搜索/列表页（路径约 `/web/geek/job`）：顶部搜索关键词框 + 城市 / 经验 / 薪资 / 学历等筛选控件 + 职位卡片列表。
- 每个职位卡片含：职位名 / 薪资区间 / 公司名 / 城市·经验·学历标签 / "立即沟通"入口。

## 筛选交互

- ⚠ 待核：筛选控件为原生 `<select>` 还是自定义下拉。若为自定义下拉，走 page-operate 时用 click 命中选项行，勿用 fill 假设已选。
- ⚠ 待核：城市/薪资等是否为级联或滑块控件；如是滑块，需以 page_snapshot 暴露的可交互 ref 为准。

## 读取

- 用 page-operate 的 `read` 步骤（带 `name` 键）采集职位卡片文本（职位名/薪资/公司/标签），回报给用户用于匹配判断，不臆造卡片未显示的信息。
